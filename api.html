API Conventions

Parameters

By default, all api methods accept the following parameters:

ignore
Number, Number[] HTTP status codes which should not be considered errors for this request.
requestTimeout
Number Milliseconds that the entire request has to complete. The default setting is inherrited from the client.
method
String The HTTP method to use for this request. All of the API methods have their own default.

Return Value/Promises

When a callback is passed to any of the API methods, it will be called with (err, response, status).The methods will also return a simple object with an abort() function. Call this method to end long running requests. Calling abort only ends the HTTP request, not the work Elasticsearch is doing.

If you prefer to use promises, don't pass a callback and a promise (with an additiional abort method) will be returned.

bulk(params, [callback])

Params:

consistency
String Explicit write consistency setting for the operation
refresh
Boolean Refresh the index after performing the operation
[replication=sync]
String Explicitely set the replication type

String Default document type for items which don't provide one

Date, Number Explicit operation timeout

String Default index for items which don't provide one

Examples

NOTE: There is a lot of great information about this endpoint in the elasticsearch docs.

Perform three operations in a single request

client.bulk({
  body: [
    { index:  { _index: 'myindex', _type: 'mytype', _id: 1 } }, // action description
    { title: 'foo' }, // the document to index
    { update: { _index: 'myindex', _type: 'mytype', _id: 2 } }, // action description
    { doc: { title: 'foo' } // the document to update
    { delete: { _index: 'myindex', _type: 'mytype', _id: 3  },  // action description
    // no document needed for this delete
  ]
}, function (err, resp, status) {
  // ...
});
