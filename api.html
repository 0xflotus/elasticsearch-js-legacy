API Conventions

Parameters

By default, all api methods accept the following parameters:

  • ignore Number, Number[] HTTP status codes which should not be treated as errors
  • requestTimeout Number Milliseconds this request has to complete. The default can be set using the client's requestTimeout: config parameter.
  • method String The HTTP method to use for this request. All of the API methods have their own default.

Return Value/Promises

When a callback is passed to any of the API methods, it will be called with (err, response, status).The methods will also return a simple object with an abort() function. Call this method to end long running requests. NOTE: Calling abort only ends the HTTP request, not the work the server is doing.

If you prefer to use promises, don't pass a callback and a promise (with an additiional abort method) will be returned.
